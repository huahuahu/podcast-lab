### 中文摘要

Dave Smith 和 Jameson Dance 在 Soft Skills Engineering 第 513 期回答两个问题：

1. **被公司"遗忘"了，还在拿工资但没人给我派活——多久后我在道德上有义务主动开口？** 听众在一家大型金融公司经历重组，从原团队被踢出后整整一个月没被重新分配，工资照发。他既享受这种摸鱼状态，又开始担心"靴子落地"时的声誉损失。Dave / Jameson 的建议：先别急着自首。绝大多数情况这是经理 / HR 的疏忽，不是你的过错；只要你愿意干活、随时可以接活就没有道德问题。可以做的：偶尔给前经理或 skip-level 发个低姿态的"need help finding next thing"邮件留个痕，把球轻轻踢回去；剩下的时间花在学习、做 side project、把自己的"被找回时的故事"准备好。真正风险不是道德，是绩效周期到了没人愿意给你写考评。

2. **13 年经验的工程师，在 AI 满天飞的世界里到底该深耕什么技能？** 这才是本集真正干货那一题。两位的答案不绕弯：写代码本身的杠杆在贬值，**系统级理解**在升值——架构层面组件怎么交互、故障点在哪、安全威胁面在哪，这些 LLM 一个人搞不定，必须有掌握上下文的人类。一个短期判断方法：看 benchmark 里**哪些还没被刷满**，没被刷满的领域多半就是 LLM 还不行、人还有价值的地方。另外，13 年的人本来就在往"少写代码、多带方向 / 多做架构"走，这条路恰好和 LLM 强项互补——他们已经习惯"用语言指挥别的工程师"，换成指挥 LLM 几乎是无缝迁移。重点放在：怎么给出高质量指令、怎么设计优秀架构、怎么真正理解系统是如何运作的。
